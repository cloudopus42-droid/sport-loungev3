# Omni Fixer Skill

> Навык универсального решения проблем в кодовой базе.

## 1. Категории фиксов
- `MISSING-IMPORT` — отсутствующие импорты (broken paths)
- `TYPESCRIPT` — несовместимости типов
- `CSS` — дубликаты, синтаксические ошибки
- `CONFIG` — неправильные конфиги (tailwind, vite)

## 2. Порядок работы
1. BugHunter сканирует первым (высокоприоритетные баги)
2. OmniFixer проверяет то, что осталось
3. Если уверенность > 90% — применяет фикс
4. Запускает сборку после каждого фикса
5. Если сборка падает — логирует и откатывает

## 3. Отличие от BugHunter
- BugHunter: UI/UX/a11y/rwd баги
- OmniFixer: импорты, типы, конфиги, инфраструктура
- Вместе покрывают 95% проблем
